What Keeps AI from Production

The Four Gaps Between Pilot and Production.

The issue is not more data or better data. It is an architectural shift: from data built for reporting to data built for reasoning. The patterns below are consistent across real estate, private equity, private credit, and infrastructure. The systems differ. The production gaps do not.

Data may be connected without being governable. Lineage may exist without the provenance an agent needs to judge trust, policy, and timing. In a fiduciary-heavy industry, that gap is not academic. It directly affects whether an answer can be acted on.

In Production

Your agent can produce answers, but not the evidence needed to stand behind them. That is where compliance and business confidence start to diverge. In practice: an LP asks what data informed a lease renewal recommendation. The agent generated it. The ERP recorded the outcome. Neither system can reconstruct the reasoning.

The same real-world entity often appears as multiple unreconciled records across systems. The agent is left reasoning across fragments rather than facts. In that environment, cross-system AI creates noise before it creates insight. Entity resolution is the prerequisite for trustworthy answers.

In Production

The same question returns different answers depending on which system the agent reaches first. In practice: an agent compares in-place rents against submarket benchmarks, but the property identifier does not match across sources. The comparison proceeds with the wrong asset matched silently.

Without effective dating, agents reason about the present using facts that were true at another moment in time. In alternative assets, that is not a minor technical issue. It affects valuation, reporting, and decision quality directly. The asset class changes. The failure mode does not.

In Production

Your agent outputs look right until someone checks the dates. Then they do not. In practice: a portfolio-level occupancy report consolidates figures from three property managers, each applying a different calculation logic. The aggregate is 91.4%. No single property actually has that occupancy.

NAV can mean different things across accounting, investment, and operations. EBITDA is often calculated differently across portfolio companies. Without governed business definitions, the agent selects one interpretation and scales it across contexts, producing outputs that may be technically coherent but operationally unreliable.

In Production

This is where technically correct outputs fail to earn operational trust. Once that confidence breaks, adoption stalls. In practice: business users stop trusting the agent after two weeks. Not because it is wrong, but because nobody agrees on what right means.
